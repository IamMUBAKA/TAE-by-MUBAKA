# MUBAKA DEPLOYMENT GOVERNANCE

Deployment governance rules:
- Netlify publish directory must be `public`
- Build command stays blank
- CLI deploy and DNS mutation are excluded from default workflow
- Public bundle contains only safe public assets

Public deploy allowed:
- public/index.html
- public/styles.css
- public/app.js

Mandatory clause:
“O-O-MUBAKA is closed only when visual posture, mechanical execution, public/private boundary, receipt truth, native exit-code verification, artifact existence verification, and founder-control authority all pass together. No isolated success state is valid outside the full cadence.”
